### Банк сценариев сбоев
Подключить LLM для генерации JSON-сценариев сбоев (disruption engine). Scenario parser маппит сценарии строго в существующие структурные переменные (по статье: blocked nodes / increased transit / risk flags): закрытый маршрут → `BLOCKED_NODE`, задержка → `INCREASED_TRANSIT`, отказ холодильника → `RISK_FLAG`/`fault_signals`, скачок спроса → event-механизм генератора (`event_multiplier`), не новый тип disruption.

### LLM-переговоры (Алг. 6)
Протокол: два агента обмениваются офферами, история H, LLM суммаризует интенты S_t = L(H), utility scores U1/U2 против порогов τ → agreement или failure за T раундов. Оффлайн-прогон поверх обученных агентов (delivery SLA-слоты — естественная пара для переговоров). Alg 7 (smart contract SLA) — блокчейн-фаза, сюда не входит.

### Lead time для inventory
Сейчас заказ пополняет склад в тот же тик — задача почти contextual bandit, прогноз обесценен.
Ввести задержку заказ→приход: очередь заказов в `GlobalState`, приход через k тиков (k = 2–3 дня, константа в `core/config.py`), продажа против прибывшего. С lead time `demand_forecast` становится необходимым: заказ под спрос дня прибытия. В Phase W очередь заменит реальный поток delivery→inventory, k — фактическое время доставки.
