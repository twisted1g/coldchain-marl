# Состояние системы — 16 июля 2026

**Фаза GenAI: контур LLM-сценариев сбоев (G4) завершён.** Построен полный цикл
Section 5 статьи: локальная LLM генерирует банк сценариев сбоев → сценарии
проигрываются в эпизодах среды → агенты оцениваются и дообучаются на устойчивость.

## Инфраструктура

- **LLM-клиент** (`llm/client.py`) — провайдер-независимый: Protocol-интерфейс +
  реализация под любой OpenAI-совместимый эндпоинт (LM Studio, Ollama, vLLM).
  Structured output через JSON Schema с retry; выявлена и обработана особенность
  локальных reasoning-моделей (ответ в `reasoning_content` при пустом `content`).
- **Схема сценария** (`llm/scenarios.py`) — 7 категорий (4 из статьи: labor strike,
  route closure, equipment malfunction, regulatory action; 3 под стресс-оси
  Fig. 37–39: heatwave, cold snap, power outage) × 6 видов эффектов в форме
  `{kind, magnitude, target_role}`. GBNF-грамматика гарантирует структуру, но не
  числовые диапазоны — их проверяет валидатор с отбраковкой и повтором.
- **Генератор банка** (`llm/generate_bank.py`) — CLI; банк сгенерирован моделью
  Nemotron 3 Nano 4B локально: **66 сценариев** (67 отбраковано валидатором,
  0 дубликатов), severity 0.32–0.87, все виды эффектов задействованы.
  Реплей всех 66 в среде — без ошибок.
- **Реплей в эпизодах** (`env/scenarios.py`) — `ScenarioRunner`: onset и целевые
  узлы/рёбра разрешаются из rng эпизода, эффекты действуют окном и откатываются;
  поверх штатного пер-тик `NoiseModel`. Включение: `config["scenario_bank"]`;
  `config["scenario_prob"]` задаёт долю эпизодов со сценарием;
  `reset(options={"scenario_id": ...})` фиксирует сценарий.
- **Стресс-оценка** (`training/marl/stress_eval.py`) — метрики блоков агентов на
  чистых эпизодах против каждой категории сценариев (held-out seed 700 000).

## Стресс-профиль baseline-агентов

Деградация согласована с физикой сценариев: temperature проседает на термальных
категориях (cold snap +15 %), spoilage — на power outage (+68 % FN), routing —
только на labor strike (+4 % return, блокировки узлов), delivery инертен (±6 %,
его рычаги сценарии не задевают).

## Дообучение с банком: доза решает

Два прогона дообучения всех агентов (warm start от baseline, 150 итераций):

| агент | prob = 1.0 (`_scn`) | prob = 0.5 (`_scn05`) |
|---|---|---|
| temperature | clean 0.207 → 0.328, регресс везде | **clean 0.158, худшая категория 0.205** |
| spoilage | power outage 0.182 → 0.247, регресс | **clean 0.068, power outage 0.087** |
| routing | **labor strike закрыт: +4 % → 0 %** | дыра осталась (+4 %) |
| inventory | без изменений (Δвесов ≤ 0.0005) | без изменений |
| delivery | без изменений (не обучаем до Phase W) | без изменений |

Ключевые выводы:

1. **Сценарий в каждом эпизоде (рецепт статьи в лоб) токсичен для непрерывного
   контроля**: temperature утратил чистый режим (catastrophic forgetting).
2. **Смешивание 50/50 действует как регуляризатор**: temperature и spoilage под
   стрессом стали лучше, чем baseline на чистых эпизодах.
3. **Дискретной навигации нужна полная доза**: уязвимость routing закрывается
   только при prob = 1.0.
4. Насыщенные политики (inventory) и необучаемые (delivery, до Phase W) к банку
   нечувствительны.

Рабочий набор модулей: `temperature_scn05`, `spoilage_scn05`, `routing_scn`,
inventory/delivery — baseline.

## Оговорки

- Стресс-метрики сняты по 3 эпизодам на сценарий; узкие метрики (fn_rate) шумны,
  для отчётных цифр нужен прогон с `--episodes 10`.
- Банк сгенерирован моделью 4B: качество текста достаточное, но разнообразие
  внутри категорий ограничено (route closure — 8 из 10, дедупликация).
- Численные ограничения схемы грамматикой не гарантируются — целостность банка
  держится на валидаторе (66/66 повторную валидацию проходят).
