# Состояние системы — 9 июля 2026

**Ядро готово.** Работают все пять агентов: temperature держит холод, routing выбирает маршруты, spoilage предсказывает порчу по графу сети, inventory управляет запасами, delivery координирует три машины. 
Награда каждого агента состоит из нескольких компонент, и веса на каждом шаге смещаются к той компоненте, которая сейчас хуже всех. А через intention buffer каждый агент видит планы остальных и подстраивается под них.

Следующий шаг - фаза GenAI. 
Сейчас агенты видят спрос только по факту, трансформер даст им прогноз вперёд, чтобы готовиться к спросу заранее. 
LLM будет генерировать сценарии сбоев (поломки, перекрытые маршруты), затем обучим систему на стрессовых ситуациях.

## Сводка по обучению

150 итераций по 40 эпизодов - все агенты сошлись:

| Агент | Return: старт → финиш |
|---|---|
| Temperature | −266 → −4.8 |
| Routing | −15.5 → **+94.5** |
| Spoilage | −12.1 → −0.09 |
| Inventory | −2.6 → −1.9 |
| Delivery (×3) | −38…−43 → −8…−9 | 

Кривые выходят на плато примерно к сотой итерации, дальше - стабильность.

Оговорка по inventory: return улучшился, но inventory cost при этом вырос с 0.35 до 0.62. 
Так в данных — вероятно, агент жертвует стоимостью хранения ради других компонент награды. Стоит разобраться при интеграции прогноза спроса.
